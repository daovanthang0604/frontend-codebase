# i18n Usage Guide

This guide explains how to use internationalization (i18n) in the DSP Backoffice application.

## Step 1: Use the `useTranslation` Hook with Namespace

In your page component, import and use the `useTranslation` hook with a namespace. **We recommend using the page name as the namespace**, but it's not mandatory.

```tsx
import { useTranslation } from "react-i18next"

export function MyPage() {
  const { t } = useTranslation("mypage")

  return (
    <div>
      <h1>{t("My Page Title")}</h1>
      <p>{t("Welcome to my page")}</p>
    </div>
  )
}
```

## Step 2: Use Plain English Text

**Important:** Write your text in plain English directly - no need to create keys manually!

✅ **DO THIS:**

```tsx
t("Audio Salad")
t("Welcome to Dashboard")
t("Create New Client")
```

❌ **DON'T DO THIS:**

```tsx
t("audioSalad")
t("welcomeToDashboard")
t("createNewClient")
```

The i18n extraction tool will automatically generate keys from your plain English text.

## Step 3: Extract Translation Keys

After writing your code with plain English text, run the extraction command:

```bash
pnpm i18n:extract
```

This command will:

- Scan your code for all `t()` function calls
- Extract the text strings
- Generate/update the translation JSON files in `public/locales/`

## Step 4: Load Namespace in Route

To ensure translations are loaded before the page renders, load the namespace in the route's `beforeLoad` function:

```tsx
import i18n from "@/utils/i18n"
import { createFileRoute } from "@tanstack/react-router"

export const Route = createFileRoute("/_authenticated/mypage")({
  component: MyPage,
  beforeLoad: async () => {
    await i18n.loadNamespaces(["mypage"])
  },
})
```
